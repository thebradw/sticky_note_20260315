## Test Case 5 — `horizontal_brewer_cellar.jpg` (completed same session)

### Additional fix made: `image_analyzer.py` — `_header_only_lane()` in isolated-header pass

**Lines ~942–968:** The isolated-header merging pass previously only forwarded labels from single-note lanes (`len(lane) == 1`). In TC5, the Cellar header note (yellow rectangular, `is_workflow_title: True`) sat in the inter-lane gap between the two swim lanes and a stray pain point landed in the same Y band, making `len(lane) == 2`. The old code classified that as a `multi_note` lane, creating an orphan Workflow 2 with no label instead of forwarding "Cellar" to the actual process-step lane.

**Fix:** Added a `_header_only_lane()` helper that returns `True` when a lane has exactly one `is_workflow_title` note and zero non-pain-point process steps. Such lanes are now included in the `isolated` list and excluded from `multi_note`. The candidate selection inside the loop was also updated to prefer the `is_workflow_title` note over the positional fallback.
